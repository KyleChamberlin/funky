# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in funky, please report it responsibly.

**Please use [GitHub Security Advisories](https://github.com/kylechamberlin/funky/security/advisories/new) to report vulnerabilities.** This allows us to assess the risk and coordinate a fix before public disclosure.

### What to include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response timeline

- **Acknowledgment:** Within 48 hours of report
- **Assessment:** Within 7 days
- **Fix or mitigation:** Within 90 days of confirmed vulnerability

### Scope

This policy applies to the latest released version of funky. We do not backport security fixes to older versions.

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | Yes       |
| Older   | No        |
